Desain Basis Data — ERD & LRS Relasional (v2)

Aplikasi Enerlyze · Smart Energy Analysis & AHP Decision Engine — Lippo Mall Puri
Pemodelan konseptual relasional: ERD → Transformasi LRS → LRS Final → Spesifikasi Tabel

Dokumen ini memetakan ulang struktur Firestore NoSQL Enerlyze menjadi model relasional. Array yang ter-embed di dalam ahp_results_enerlyze (yaitu ranking dan criteria) dinormalisasi menjadi tabel tersendiri agar memenuhi kaidah basis data relasional.

01Entity Relationship Diagram (ERD)

Model Enerlyze terdiri atas enam entitas utama. USER adalah penanggung jawab banyak AREA sekaligus penginput banyak ENERGY_DATA. Tiap AREA memiliki banyak catatan ENERGY_DATA (satu per bulan). AHP_RESULT adalah hasil analisis per bulan; satu hasil memeringkat banyak AREA dan menimbang keempat CRITERIA — dua hubungan ini bersifat many-to-many. NOTIFICATION ditujukan ke satu USER.

02Transformasi ERD ke LRS

Logical Record Structure (LRS) diperoleh dengan menerapkan aturan transformasi pada ERD di atas. Berikut keputusan transformasi yang diambil:

  • One-to-Many USER → AREA: foreign key user_id ditempatkan pada tabel sisi many, yaitu AREA.
  • One-to-Many AREA → ENERGY_DATA dan USER → ENERGY_DATA: foreign key area_id dan created_by ditempatkan pada ENERGY_DATA (sisi many).
  • One-to-Many USER → NOTIFICATION: foreign key user_id ditempatkan pada NOTIFICATION.
  • Many-to-Many AHP_RESULT ↔ AREA: menghasilkan tabel junction baru AHP_RANKING dengan primary key gabungan (ahp_id + area_id). Atribut relasi — score, rank_position, serta rincian kontribusi tiap kriteria — ikut ke tabel junction ini.
  • Many-to-Many AHP_RESULT ↔ CRITERIA: menghasilkan tabel junction baru AHP_RESULT_CRITERIA dengan primary key gabungan (ahp_id + criteria_key). Atribut relasi weight dan rank_used (bobot & rank kriteria pada saat analisis) disimpan di tabel ini.
  • Atribut multivalued breakdown (kontribusi 4 kriteria pada skor tiap area) ditransformasi menjadi empat kolom eksplisit pada AHP_RANKING: breakdown_energy, breakdown_cost, breakdown_duration, breakdown_maintenance — karena jumlah kriteria tetap (4).

Hasil transformasi menghasilkan 8 tabel relasional:
